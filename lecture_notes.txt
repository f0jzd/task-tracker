Array och If-satser

.push = sätter till ett element i slutet av arrayen
.pop = tar bort elementet på slutet av arrayen, tar bort och är inte en referens till slutet
.shift och .unshift gör samma fast i börjanm av arrayen, unshift lägger till, shift tar bort. 

for loop för index, när behöver hålla koll på varven
for of när man ska gå igenom en array, för varje task i array t.ex. visa alla uppgifter
foreach när man vill göra något med varje element, ska visa alla, ska köra samma kodsnutt på varje
while är bra när man inte vet hur många gånger loopen ska köra. 
do while samma som while, men göra en sak minst en gång.

Type/ Interface

type = inte data, bara beskrivning av data
ts använder beskrivningen för att hitta fel.

type Status = "Pending" | "Completed" => kan bara vara dem två strängarna

union types = ett av flera bestämda val

Interface = kontrakt, beskriver exakt vilka properties ska finnas i objektet, något man måste följa
type = allt möjligt hit och dit
Interface = objekt, kontrakt för objekt, hur de ska vara, mer styrt och bestämt

type vs interface välj det som passar situationen, förstå båda och följ arbetsplatsen.

funktion i objekt = metod

typning gör saker mer klart och tydligt. typ som kommentarer i kod

unions försvinner när typescript komplieras, enums do not. 
union enklare, enums värden med namn
